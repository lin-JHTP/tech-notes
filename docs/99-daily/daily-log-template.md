#### 🧠 今天学到了什么

**技术知识：**

1. **FreeRTOS 队列 API 细节**
   - `xQueueSend()` vs `xQueueOverwrite()`：前者在满时阻塞，后者在满时覆盖
   - 在 ISR 中必须用 `xQueueSendFromISR()`，否则系统崩溃
   - 推荐的队列大小：预期最大缓存量的 2~3 倍

2. **OCPP StartTransaction 关键点**
   - `meterStart` 字段使用能量计初始电表读数（Wh），不是功率
   - CSMS 返回的 `transactionId` 需要本地持久化，断电重启后还需用到
   - 离线时应本地缓存事务，恢复后上传

3. **UART IDLE 中断调试经验**
   - GD32 IDLE 标志清除需要先读 SR，再读 DR（一对操作）
   - 仅读 SR 或仅读 DR 都不能清除 IDLE 标志
   - 参考：[串口与DMA调试经验](../01-embedded/uart-dma-debug.md)

---

#### ❓ 遇到的问题

**问题1：FreeRTOS 队列偶发死锁**

- **现象**：系统运行约 30 分钟后，UART 接收任务无响应
- **排查过程**：
  1. 用 `vTaskList()` 打印任务列表，发现 UART_RX 任务处于 Blocked 状态
  2. 检查发现主处理任务长时间持有互斥锁后异常退出，未释放
  3. 互斥锁永久被占用，等待的任务永久阻塞
- **解决方案**：所有持锁路径（包括错误处理路径）都确保释放锁
- **教训**：互斥锁使用 `goto cleanup` 模式统一释放

## 💡 记录技巧

1. **不要追求完美**：哪怕只写两三句话也比不写强
2. **问题要记录解决过程**：光写"解决了"没价值，写清楚怎么找到原因的
3. **想法及时记下来**：好想法稍纵即逝，先记再说
4. **明天计划要具体**：不要写"继续学习"，写"完成 XXX 功能的 YYY 部分"
