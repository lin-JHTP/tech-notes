## 1. 进程与线程

### 核心区别

| 对比项 | 进程 | 线程 |
|--------|------|------|
| 定义 | 程序的一次执行实例 | 进程内的执行单元 |
| 资源 | 独立的地址空间、文件描述符 | 共享进程的地址空间和资源 |
| 切换开销 | 大（需切换地址空间） | 小（只切换寄存器/栈） |
| 通信 | IPC（管道/信号/共享内存） | 直接共享内存（需同步） |
| 崩溃影响 | 不影响其他进程 | 一个线程崩溃可能影响整个进程 |
| 创建开销 | 大（fork + exec） | 小（pthread_create） |

### 进程状态机

```
新建(New)
    │ 创建完成
    ▼
就绪(Ready) ←─────────── 等待事件发生
    │ 调度器选中
    ▼
运行(Running) ──────────→ 等待 I/O/锁
    │ 进程结束          阻塞(Blocked)
    ▼
终止(Terminated)
```

### 进程间通信（IPC）

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 管道（pipe） | 简单，有内核缓冲 | 单向，只能父子进程 | Shell 管道 |
| 命名管道（FIFO） | 支持无关进程 | 仍是单向 | 本地进程通信 |
| 消息队列 | 有结构，有类型 | 大小限制 | 异步消息传递 |
| 共享内存 | 速度最快（零拷贝） | 需要同步原语 | 高性能数据共享 |
| 信号量 | 同步原语 | 不适合传数据 | 进程同步 |
| Socket | 跨主机 | 开销较大 | 网络/跨机通信 |
| 信号（Signal） | 异步通知 | 信息量少 | 异常通知 |

## 2. 进程调度算法

| 算法 | 全称 | 特点 | 优点 | 缺点 |
|------|------|------|------|------|
| FCFS | 先来先服务 | 非抢占 | 简单公平 | 护航效应（长任务阻塞短任务） |
| SJF | 最短作业优先 | 可抢占/非抢占 | 平均等待时间最短 | 需要预知执行时间，饥饿问题 |
| RR | 轮转调度 | 抢占（时间片） | 公平，适合分时系统 | 上下文切换开销 |
| Priority | 优先级调度 | 可抢占 | 重要任务优先 | 饥饿（低优先级任务长期得不到调度） |
| MLFQ | 多级反馈队列 | 抢占 | 兼顾响应和吞吐 | 实现复杂 |

### Linux 的 CFS（完全公平调度器）

Linux 2.6.23+ 使用 CFS，核心思想：

- 每个进程维护**虚拟运行时间（vruntime）**
- vruntime = 实际运行时间 × (1 / 权重)
- 调度器始终选择 vruntime 最小的进程运行
- 用红黑树维护所有就绪进程，O(log n) 选择

## 3. 内存管理

### 虚拟内存

每个进程有独立的**虚拟地址空间**，通过页表映射到物理内存：

```
虚拟地址空间（32位：0~4GB）
┌──────────────────┐ 0xFFFFFFFF（4GB）
│   内核空间（1GB）  │
├──────────────────┤ 0xC0000000（3GB）
│   用户栈           │ ↓ 向下增长
│   ...              │
│   共享库           │
│   ...              │
│   堆               │ ↑ 向上增长
│   BSS（未初始化）  │
│   数据段（已初始化）│
│   代码段（只读）   │
└──────────────────┘ 0x00000000（0）
```

### 页面置换算法

| 算法 | 描述 | 特点 |
|------|------|------|
| OPT（最优） | 置换未来最久不用的页 | 最优但不可实现（需预知未来） |
| FIFO | 先进先出置换 | 简单，可能有 Belady 异常 |
| LRU | 最近最久未使用 | 接近最优，开销较大 |
| Clock | LRU 近似算法 | 较低开销，实际广泛使用 |
| LFU | 最不常用 | 考虑访问频率 |

### 内存分配

```
堆内存分配算法：
- 首次适配（First Fit）：找第一个够大的空闲块，速度快
- 最佳适配（Best Fit）：找最小够大的块，碎片小但速度慢
- 最差适配（Worst Fit）：找最大块，减少碎片但降低大分配成功率
- 伙伴系统（Buddy）：Linux 内核使用，分配/合并效率高
- SLAB/SLUB：内核对象缓存，减少小对象分配开销
```

## 4. 死锁

### 死锁的四个必要条件（Coffman 条件）

1. **互斥**：资源不能共享（一次只能一个进程使用）
2. **占有并等待**：进程持有已有资源，又在等待新资源
3. **不剥夺**：已获得的资源不能被强制剥夺
4. **循环等待**：存在进程→资源的循环等待链

**破坏任意一条即可预防死锁。**

### 银行家算法（死锁避免）

```
状态：安全状态（存在安全序列）→ 可以分配
      不安全状态（不存在安全序列）→ 拒绝分配

安全序列：能找到一个进程执行顺序，使所有进程都能完成
```

### 实际解决方案

| 方法 | 实现方式 | 代价 |
|------|----------|------|
| 预防 | 破坏四个条件之一 | 可能导致资源利用率低 |
| 避免 | 银行家算法 | 每次分配都要计算，开销大 |
| 检测+恢复 | 定期检测，发现后强制终止 | 可能丢失工作 |
| 忽略 | 鸵鸟算法（Linux/Windows使用） | 简单，死锁时重启进程 |

## 5. 文件系统基础

### inode（索引节点）

Linux 文件系统中，每个文件由一个 inode 描述：

| inode 字段 | 内容 |
|-----------|------|
| 文件类型 | 普通文件/目录/软链接等 |
| 权限 | rwxrwxrwx |
| 链接数 | 硬链接计数 |
| 所有者 UID/GID | |
| 文件大小 | |
| 时间戳 | atime/mtime/ctime |
| 数据块指针 | 直接/间接/双重间接块 |

**硬链接 vs 软链接：**

| 对比项 | 硬链接 | 软链接（符号链接） |
|--------|-------|-----------------|
| 本质 | 同一 inode 的不同目录项 | 指向路径的特殊文件 |
| 跨文件系统 | ❌ 不支持 | ✅ 支持 |
| 目录链接 | ❌ 不支持 | ✅ 支持 |
| 源文件删除 | 链接仍可用 | 链接失效（悬空链接） |

## 6. I/O 模型

Linux 的五种 I/O 模型（以 socket read 为例）：

| 模型 | 描述 | 阻塞阶段 |
|------|------|----------|
| 阻塞 I/O | 调用 read，等到数据到达后返回 | 等待数据 + 数据拷贝 |
| 非阻塞 I/O | 轮询 read，无数据立即返回 EAGAIN | 只在数据拷贝时阻塞 |
| I/O 多路复用 | select/poll/epoll 等待多个 fd | 等待就绪（select阻塞）+ 数据拷贝 |
| 信号驱动 I/O | SIGIO 信号通知就绪 | 只在数据拷贝时阻塞 |
| 异步 I/O | aio_read，完成后通知 | 完全不阻塞 |

### epoll vs select/poll

| 对比 | select/poll | epoll |
|------|-------------|-------|
| 最大 fd 数 | select: 1024；poll: 无限制 | 无限制 |
| 时间复杂度 | O(n)（每次遍历所有 fd） | O(1)（事件驱动） |
| 内存拷贝 | 每次调用都拷贝 fd 集合 | 一次注册，无重复拷贝 |
| 工作模式 | 水平触发（LT） | LT + 边缘触发（ET） |
| 适用场景 | 少量 fd | 高并发服务器（C10K 问题） |

## 7. 高频面试问题汇总

**Q：进程和线程的区别？**
> A：进程是资源分配的基本单位（独立地址空间），线程是 CPU 调度的基本单位（共享进程资源）。线程切换开销更小，但需要同步避免竞争。

**Q：为什么要有虚拟内存？**
> A：① 地址隔离，进程之间无法直接访问对方内存（安全）；② 内存扩展，可以使用超过物理内存的地址空间（通过换页）；③ 简化编程模型，每个进程都看到完整的地址空间。

**Q：什么是死锁？如何避免？**
> A：死锁是多个进程相互等待对方持有的资源导致都无法继续。避免方式：按固定顺序申请锁、使用超时锁、银行家算法。

**Q：select、poll、epoll 的区别？**
> A：select 有 fd 数量限制（1024），O(n) 遍历；poll 无数量限制但仍 O(n)；epoll 基于事件驱动，O(1) 获取就绪 fd，适合高并发场景。

**Q：用户态和内核态的区别？**
> A：内核态可以执行所有 CPU 指令、访问所有内存；用户态只能执行受限指令，访问受限内存。系统调用是从用户态切换到内核态的入口（软中断或 SYSCALL 指令）。
