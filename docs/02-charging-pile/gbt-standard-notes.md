## 1. 中国充电标准体系概览

```
中国充电标准
├── 接口标准
│   ├── GB/T 20234.1-2023  交流充电接口
│   ├── GB/T 20234.2-2015  直流充电接口
│   └── GB/T 20234.3-2015  无线充电接口
├── 通信协议
│   ├── GB/T 27930-2023    直流充电通信协议（桩-车，基于 CAN）
│   └── GB/T 34658-2017    互联互通协议（桩-平台，基于 TCP）
├── 系统规范
│   ├── GB/T 18487.1-2023  传导充电系统总则
│   └── NB/T 33001-2022    充电桩技术规范
└── 运营标准
    └── GB/T 40032-2021    充电运营管理规范
```

## 2. GB/T 27930 协议要点

GB/T 27930 是**直流充电桩与电动汽车（BMS）之间**的通信协议，基于 CAN 总线。

### CAN 总线参数

| 参数 | 值 |
|------|----|
| 波特率 | 250 kbps |
| 帧格式 | 标准帧（11位ID）|
| 接口 | 充电枪 PIN5/PIN6（CAN-H/CAN-L）|

### 充电握手流程（九阶段）

```
BHM（充电机握手报文）← EVSE 主动发
    │
    ▼
BRM（电池管理系统和车辆辨识报文）← BMS 应答
    │
    ▼
BCP（电池充电参数报文）← BMS 发送充电参数
    │
    ▼
CML（充电机最大输出能力报文）← EVSE 发送能力
    │
    ▼
BRO（电池充电准备就绪报文）← BMS 确认准备好
    │
    ▼
CRO（充电机输出准备就绪报文）← EVSE 确认
    │
    ▼
BCL（电池充电需求报文）← BMS 实时需求
    │
    ▼ （充电中循环）
BCS（电池充电状态报文）← BMS 实时状态
CCS（充电机充电状态报文）← EVSE 实时状态
    │
    ▼
BST（电池中止充电报文）← BMS 请求停止
CST（充电机中止充电报文）← EVSE 确认停止
    │
    ▼
BSD（电池统计数据报文）
CSD（充电统计数据报文）
    │
    ▼
BEM（错误报文）/ CEM（错误报文）
```

### 关键报文字段（BCP - 电池充电参数）

| 字段 | 含义 | 字节 |
|------|------|------|
| 单体电池最高允许充电电压 | BMS 允许的单体最大电压 | 2 |
| 最高允许充电电流 | 允许的最大充电电流（绝对值） | 2 |
| 动力蓄电池标称总能量 | 电池包额定能量 | 2 |
| 最高允许充电总电压 | 电池包最大允许电压 | 2 |
| 最高允许温度 | 充电允许的最高温度 | 1 |
| 整车动力蓄电池当前荷电状态 | 当前 SOC（×0.1%） | 2 |
| 当前充电状态 | 初次/继续充电 | 1 |

## 3. OCPP vs GB/T 27930 对比

| 对比维度 | OCPP 1.6/2.0.1 | GB/T 27930-2023 |
|----------|-----------------|-----------------|
| **适用范围** | 充电桩 ↔ 充电管理系统 | 充电桩 ↔ 电动汽车BMS |
| **传输层** | WebSocket/TCP | CAN 总线 |
| **消息编码** | JSON | 二进制（CAN 帧） |
| **发起方** | 充电桩主动连接 CSMS | 充电桩/BMS 按规范时序 |
| **主要内容** | 计费、认证、远程控制 | 充电参数协商、充电控制 |
| **实时性** | 秒级（网络延迟） | 毫秒级（CAN） |
| **国际互认** | 全球通用 | 中国特有 |
| **版本管理** | OCA 维护 | 国家标准，定期修订 |
| **安全机制** | TLS、证书 | 无（依赖物理连接） |

### 在充电桩中的角色

```
充电管理平台 (CSMS)
        │
        │  OCPP（WebSocket）
        │  负责：认证、计费、远程控制、运营
        ▼
充电桩 (EVSE)
        │
        │  GB/T 27930（CAN 总线）
        │  负责：充电参数协商、充电过程控制
        ▼
电动汽车 BMS
```

两套协议**并行运行**，分别负责不同层面：

- OCPP：商务/运营层（谁在充？收多少钱？什么时候停？）
- GB/T 27930：物理/控制层（以什么电流电压充？现在电池状态如何？）

## 4. 互联互通标准 GB/T 34658

GB/T 34658 解决不同运营商平台之间的互联互通问题（跨网充电）。

### 协议栈

- 传输层：TCP（JSON over HTTP 或裸 TCP）
- 使用 MQTT 或 HTTP RESTful 变体
- 通常由聚合平台（如国家互联互通平台）实现

### 核心功能

| 功能 | 说明 |
|------|------|
| 设备信息同步 | 充电桩位置、状态、功率信息 |
| 实时状态推送 | 充电桩在线/空闲/充电中 |
| 充电认证 | 跨网卡/APP 授权 |
| 计费结算 | 跨平台订单和费用清算 |
| 统计报表 | 充电量、用电量统计上报 |

## 5. 工程注意点与踩坑记录

### GB/T 27930 实现踩坑

**坑1：充电握手超时**

> 各阶段握手有严格的超时要求。BMS 侧超时未响应，充电桩必须中止。
> 容易犯的错误：超时处理漏掉了，导致充电桩永久等待。
> 
> 解决：为每个握手阶段加独立超时计时器。

**坑2：充电电流爬坡**

> 不能从 0 直接跳到目标电流，必须平滑爬坡（通常 1A/s 或按 BMS 要求）。
> 急速变化可能触发 BMS 过流保护或充电桩保护。

**坑3：报文字节序（大端/小端）**

> GB/T 27930 使用**小端字节序**（低字节在前）。
> 注意：与网络协议（大端）相反，容易犯错。

**坑4：SOC 值含义**

> BCP 报文中的 SOC 单位是 **0.1%**，即值为 850 表示 85.0% SOC。
> 转换时注意乘以 0.1。

### OCPP 与 GB/T 27930 联动注意点

| 事件 | OCPP 动作 | GB/T 27930 动作 |
|------|-----------|-----------------|
| 用户刷卡授权 | Authorize → StartTransaction | 等待，授权后才发 CRO |
| 充电中远程停止 | RemoteStopTransaction | 发送 CST 中止充电 |
| BMS 请求停止 | StopTransaction（reason: EVDisconnected） | 接收 BST 后发 CST |
| 网络断线 | 本地记录，恢复后上传 | 不影响，继续充电 |
