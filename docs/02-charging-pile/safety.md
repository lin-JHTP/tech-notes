# 充电桩安全机制

> 充电安全是充电桩设计的第一优先级，本文汇总交流慢充桩的主要安全保护机制。

## 📌 安全保护项汇总

| 保护类型 | 检测方式 | 触发动作 |
|---------|---------|---------|
| 漏电保护（RCD） | 剩余电流互感器 | 立即断开主回路 |
| 过流保护 | 计量芯片 / 电流互感器 | 断开接触器 |
| 过压 / 欠压 | ADC 采样 | 停止充电，上报故障 |
| 过温保护 | NTC 温度传感器 | 降功率或停止充电 |
| 绝缘检测 | 绝缘检测电路 | 阻值低于阈值则禁止充电 |
| CP 信号异常 | 比较器 / ADC | 停止充电 |
| 接触器粘连检测 | 继电器反馈 / 电压检测 | 上报故障，禁止下次充电 |

---

## 🔑 漏电保护（RCD）

交流慢充桩**强制要求** A 型或 B 型剩余电流保护装置（RCD）。

- **A 型**：保护交流和脉动直流漏电（最低要求）
- **B 型**：同时保护平滑直流漏电（更高要求）
- 动作电流：≤30mA（人身安全阈值）
- 响应时间：≤25ms

---

## 🔑 CP 信号保护逻辑

CP 信号异常是最常见的故障触发原因：

```c
/* CP 电压状态枚举 */
typedef enum {
    CP_STATE_A = 0,   /* 12V：无车 */
    CP_STATE_B,       /* 9V：车连接 */
    CP_STATE_C,       /* 6V：车就绪 */
    CP_STATE_E,       /* 0V：错误 */
} CPState;

/* CP 检测示例（简化） */
CPState get_cp_state(uint16_t adc_value)
{
    if (adc_value > 3500)      return CP_STATE_A;  /* ~12V */
    else if (adc_value > 2800) return CP_STATE_B;  /* ~9V */
    else if (adc_value > 2000) return CP_STATE_C;  /* ~6V */
    else                       return CP_STATE_E;  /* 异常 */
}
```
